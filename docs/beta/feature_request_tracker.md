# Beta Feature Request Tracker

This document aggregates net-new feature requests submitted by beta testers. Features will be evaluated based on feasibility, alignment with the product roadmap, and tester demand (upvotes).

## Prioritization Matrix
- **Must Have (Phase 9):** Critical for public commercial launch.
- **Should Have (Post-Launch):** High value, but non-blocking for Phase 9.
- **Could Have (Backlog):** Niche or edge-case requests.
- **Won't Have (Rejected):** Out of scope for FiduScan's vision.
