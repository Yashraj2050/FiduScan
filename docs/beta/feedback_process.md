# Feedback Collection System

## Overview
A structured system for capturing, triaging, and actioning insights generated during the Closed Beta.

## Tracking Categories
1. **Bugs:** Technical failures, 500 errors, broken UI components, or inaccurate analysis results.
2. **UX Issues:** Workflows that require too many clicks, unintuitive navigation, or confusing copy.
3. **Feature Requests:** Missing capabilities requested by testers (e.g., "Need bulk export").
4. **Confusion Points:** Areas where testers required support or external documentation to proceed.

## Data Collection Channels
- **In-App Widget:** A persistent "Give Feedback" button on the UI that captures screen state and console logs.
- **Async Community:** Slack channel (`#fiduscan-beta-feedback`).
- **Telemetry:** Automated tracking of active users, investigations created, reports generated, evidence records created, and exports generated.
